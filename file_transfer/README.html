File Transfer Automation

Drop a file into a watched folder → it gets routed automatically to Slack, Teams, WhatsApp, Telegram, Zoom, OneDrive, Box, SFTP, WeTransfer, or a local backup — based on rules you define.

How It Works

The tool runs a continuous loop watching a folder. New files are matched against rules (top-down, first match wins) and dispatched to one or more handlers in sequence.

┌──────────────┐ ┌──────────────┐ │ inbox/ folder│──file─▶│ Watcher │ └──────────────┘ └──────┬───────┘ │ match rules ┌──────┬──────┬──────┼──────┬──────┬──────┐ ▼ ▼ ▼ ▼ ▼ ▼ ▼ Slack Teams Telegram Zoom OneDrive Box WhatsApp SFTP Local WeTransfer

Each handler is a self-contained Python class. Credentials come from .env; routing logic lives in config.yaml.

Prerequisites

  • Python 3.10+ — verify with python --version
  • pip — comes with Python; used to install dependencies
  • Service credentials — only for the handlers you plan to use (see each handler's setup section below)

Installation

  1. Install Python packages 
    pip install -r requirements.txt
  2. Create your environment file 
    # PowerShell
copy .env.example .env
  3. Edit .env — fill in credentials for the services you want to use. You only need to configure the handlers you enable in config.yaml.
  4. (Optional) Customise config.yaml — add, remove, or reorder routing rules.

Quick Run

powershell# Start the watcher (Ctrl+C to stop)
python main.py

The watch folder (default inbox\) is created if missing. Drop a file in there and watch the logs.

💡 Tip
To test quickly without any external credentials, enable only the local_copy handler.

Handler: Slack

Uploads files to a Slack channel or DM using the Slack Bot API.

Setup

  1. Go to api.slack.com/apps → Create New App → From Scratch.
  2. Under OAuth & Permissions, add scopes: files:write, chat:write.
  3. Install to Workspace and copy the Bot User OAuth Token (xoxb-…).
  4. Set SLACK_BOT_TOKEN and SLACK_CHANNEL in .env.
  5. Invite the bot to the channel: /invite @YourBotName.

Rule options

KeyRequiredDescription
typeYesMust be slack
messageNoText sent with the file

Handler: Microsoft Teams

Uploads files to a Teams channel via the Microsoft Graph API, then posts a link message.

Setup

  1. In the Azure Portal → App Registrations, create a new registration.
  2. Under API Permissions, add Application permissions: ChannelMessage.Send, Files.ReadWrite.All, ChannelSettings.Read.All. Grant admin consent.
  3. Under Certificates & secrets, create a client secret.
  4. Copy the Tenant ID, Client ID, and Client Secret into .env.
  5. Get the Team ID and Channel ID from Teams (channel link URL or Graph API explorer) and set them in .env.

Rule options

KeyRequiredDescription
typeYesMust be teams
messageNoText posted with the file link

Handler: WhatsApp

Sends files via the Twilio WhatsApp Business API. Twilio fetches the file from a public URL you provide.

Setup

  1. Sign up at twilio.com/console and note your Account SID and Auth Token.
  2. Activate the WhatsApp Sandbox (or register a WhatsApp Business number).
  3. Set WHATSAPP_FROM_NUMBER (e.g. whatsapp:+14155238886) and WHATSAPP_TO_NUMBER.
  4. Provide a WHATSAPP_MEDIA_BASE_URL — a publicly reachable URL where the watcher's files are available (e.g. via ngrok pointing to the inbox folder).
⚠️ Media URL
Twilio must be able to fetch the file over HTTP. You need to serve the inbox folder via a simple HTTP server or tunnel. Example: npx http-server ./inbox -p 8080 + ngrok.

Rule options

KeyRequiredDescription
typeYesMust be whatsapp
messageNoText sent with the file

Handler: Telegram

Sends files as documents to a Telegram chat or group using the Bot API.

Setup

  1. Open Telegram and message @BotFather. Run /newbot and follow the prompts to create a bot. Copy the token.
  2. Send any message to your new bot, then visit https://api.telegram.org/bot<TOKEN>/getUpdates in a browser to find the chat ID.
  3. Set TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID in .env.

Rule options

KeyRequiredDescription
typeYesMust be telegram
messageNoCaption sent with the file

Handler: Zoom

Shares files in a Zoom Chat channel via the Zoom Server-to-Server OAuth API.

Setup

  1. Go to the Zoom App Marketplace → Develop → Build App → Server-to-Server OAuth.
  2. Add scopes: chat_message:write, chat_file:write.
  3. Activate the app and copy Account ID, Client ID, and Client Secret.
  4. Find the target Chat Channel ID via the Zoom API (GET /chat/users/me/channels).
  5. Set all four values in .env.

Rule options

KeyRequiredDescription
typeYesMust be zoom
messageNoCompanion text message

Handler: OneDrive

Handler: WeTransfer

Automates the WeTransfer web UI via Playwright to upload a file and email a download link to the configured recipient. WeTransfer requires email verification — the handler auto-fetches the code via MailSlurp.

Setup

  1. Install Playwright browsers after pip install: 
    python -m playwright install chromium
  2. Create a free MailSlurp account and create an inbox. Copy the API Key and the inbox email address.
  3. Set WETRANSFER_SENDER_EMAIL to the MailSlurp inbox email (e.g. monitor@mailslurp.biz).
  4. Set WETRANSFER_RECIPIENT_EMAIL — who receives the file (can be the same MailSlurp address).
  5. Set MAILSLURP_API_KEY — allows the handler to auto-fetch the WeTransfer verification code.
  6. (Optional) Set WETRANSFER_HEADLESS=false to watch the browser during debugging.
⚠️ Email Verification
WeTransfer requires a verification code sent to the sender's email. The handler uses MailSlurp to fetch this code automatically. Without MAILSLURP_API_KEY, the transfer will fail.

Rule options

KeyRequiredDescription
typeYesMust be wetransfer
messageNoTransfer description / message text

Troubleshooting

Nothing happens when I drop a file

  • Verify you see "Watching folder: …" in the console.
  • Confirm the file is going into the correct WATCH_FOLDER.
  • Set LOG_LEVEL=DEBUG in .env for verbose output.

"handler has invalid credentials – skipping it"

⚠️
Required env vars for that handler are missing. Check .env against the table above.

Slack: not_in_channel

Invite the bot: /invite @YourBotName in the target channel.

Teams: 401 Unauthorized

Ensure admin consent was granted for the API permissions in Azure AD.

WhatsApp: File not received

WHATSAPP_MEDIA_BASE_URL must be publicly reachable. Test by opening the URL in a browser.

Telegram: chat not found

You must message the bot first so it has a chat ID. For groups, add the bot to the group and use the group chat ID (usually a negative number).

Zoom: 403 Forbidden

Verify the app has the chat_message:write and chat_file:write scopes and is activated.

Cloud storage: files not syncing

Verify the desktop client (OneDrive / Box Drive) is running and signed in. Confirm the sync folder path in .env matches the actual folder the client is syncing.

SFTP: Connection refused

  • Confirm SFTP_HOST and SFTP_PORT.
  • If using a private key, ensure it's OpenSSH/PEM format.

WeTransfer: upload fails or times out

  • Set WETRANSFER_HEADLESS=false and run again to see what the browser is doing.
  • WeTransfer may show a cookie consent or paywall — the handler tries to dismiss these, but UI changes may break selectors.
  • Ensure playwright install chromium has been run after installing requirements.
